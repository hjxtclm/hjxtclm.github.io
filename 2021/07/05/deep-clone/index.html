JS 浅拷贝和深拷贝

2024-08-19 · 3 min · 812 words · JS 基础知识

浅拷贝

浅拷贝是创建一个新对象，复制源对象的引用，而不是复制其内容。这意味着源对象包含引用类型的属性时（如对象和数组），目标对象和源对象会共享相同的引用，改变其中一个会影响到另一个。

深拷贝

深拷贝会创建一个全新的对象，不仅复制源对象的属性，二球递归复制所有嵌套的对象和数组，确保它们是独立的副本，改变深拷贝后的对象不会影响源对象。

JSON.parse(JSON.stringify(obj))

限制：

  1. 忽略 undefined: JSON 格式不支持 undefined 值，所以序列化时，任何 undefined 属性都会丢失。

  2. 忽略 symbol: JSON 格式不支持 Javascript 的 symbol 类型，所以 symbol 类型会被忽略

  3. 不能处理函数：JSON 格式不支持函数，所以函数会被忽略。

  4. 不能处理循环引用：如果对象中包含循环引用，JSON.stringify 会抛出错误。

  5. 日期对象：日期对象会被转换成字符串，失去了原有的日期类型

  6. 正则表达式对象：正则表达式对象不会被序列化，而是被转换为 {}

  7. 对象的原型链信息：JSON.stringify 只能处理对象的自身属性，不会保留原型链信息。

  8. 性能问题：对于非常大的或者复杂对象，这种方法可能会非常慢，因为它需要序列化整个对象，然后再解析回一个新的对象。

  9. 精度损失：当处理浮点数时，由于 Javascript 的浮点数表示的精度问题，可能会在序列化和反序列化过程中造成轻微的精度损失。

手动创建对象并复制属性

其实实在浅拷贝的基础上，添加递归和数据类型判断。

function deepClone(obj, hash = new WeakMap()) {
  // 非 object 或者 null，直接返回
  if (obj === null || typeof obj !== 'object') {
    return obj;
  }

  // 检测循环引用
  if (hash.has(obj)) {
    return hash.get(obj);
  }

  // 下面的逻辑用于处理各种特殊的数据类型，如需要处理更多类型，可以在此处添加
  let copy

  if (Array.isArray(obj)) {
    // 数组类型
    copy = []
    hash.set(obj, copy)
    for (let i = 0; i < obj.length; i++) {
      // 递归复制
      copy[i] = deepClone(obj[i], hash)
    }
  } else if (obj instanceof Date) {
    // 处理日期对象
    copy = new Date(obj.getTime())
  } else if (obj instanceof RegExp) {
    // 处理正则表达式对象
    copy = new RegExp(obj)
  } else {
    // 处理普通对象
    // 通过 new obj.constructor() 创建一个与 obj 相同类型的对象
    copy = new obj.constructor()
    hash.set(obj, copy)
    for (let key in obj) {
      if (obj.hasOwnProperty(key)) {
        // 递归复制
        copy[key] = deepClone(obj[key], hash)
      }
    }
  }

  return copy
}
