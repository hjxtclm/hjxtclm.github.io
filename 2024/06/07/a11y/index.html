前端无障碍介绍

什么是前端无障碍

无障碍（Accessibility，常被缩写为 A11y，因为它以“a”开头，后跟 11 个字母，以“y”结尾）在 web 开发中意味着使尽可能多的人能够使用网站，即使有些用户的能力在某种程度上受限。

指在⽹⻚设计和开发过程中，采⽤⼀系列技术和标准，使⽹站能够被残障⼈⼠访问和使⽤，从⽽实现⽹站的⽆障碍访问。

前端无障碍的重要性及发展流程

前端⽆障碍的重要性在于，它能够让残障⼈⼠也能够访问和使⽤⽹站，从⽽提⾼⽹站的⽤⼾体验，提升⽹站的可⽤性，提⾼⽹站的流量，从⽽达到更好的商业效果。

前端⽆障碍的发展历程可以追溯到 20 世纪 90 年代，当时美国政府颁布了《美国⽆障碍法案》，要求政府⽹站必须实现⽆障碍访问，从⽽开启了前端⽆障碍的发展历程。随着科技的发展，前端⽆障碍也在不断演进，如今，已经有了⼀系列的标准和技术，可以帮助开发者实现⽆障碍访问。

无障碍优化开发

读屏软件

⽆障碍访问的关键点⸺使⽤屏幕阅读器。

移动端 APP 访问⽆障碍特性，开启读屏模式的设置路径：

  • iOS： 设置 -> 通⽤ -> 辅助功能 -> 旁⽩（VoiceOver） 。

  • 安卓： 设置 -> 辅助功能 -> ⽆障碍 -> TalkBack （不同机型路径可能不⼀样）。

⼩⽶⼿机可通过同时按住加减⾳量键打开和关闭TalkBack（未测试其他品牌⼿机⽀不⽀持）

无障碍手势操作（安卓）

  • 轻点：选择并朗读项⽬

  • ⼿指按住屏幕向上或向下：按顺序读取屏幕元素

  • 轻点两下：激活所选项⽬

  • 左右轻扫：选择上⼀项或下⼀项

  • 双指上下轻扫：滚动屏幕

WAI-ARIA

WAI-ARIA 通过浏览器把信息暴露给 accessibility APIs (⽆障碍API)，作为读屏软件的信息源。
WAI-ARIA 是 W3C 编写的规范，定义了⼀组可⽤于其他元素的 HTML 特性，⽤于提供额外的语义化以及改善缺乏的⽆障碍。以下是规范中三个主要的特性

特性 描述 示例
⻆⾊ 定义元素是⼲什么的 例如：
role=“tab”
role=“button”
role=“radio”
role=“checkbox”
role=“link”
role=“alert”
属性 让元素具备更多语义 例如：
aria-haspopup
aria-label
aria-readonly
状态 ⽤于表达元素当前的条件的特殊属性 例如：
aria-checked
aria-selected
aria-expanded
aria-hidden
aria-invalid
